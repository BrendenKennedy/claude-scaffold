# reference — stable supporting notes

Tool- or topic-specific notes that should stay **consistent across sessions** but don't warrant a
full skill. This is the "how do we do X in this repo" shelf.

- **Belongs here:** CLI cheat-sheets, repo workflow conventions, library gotchas, small how-tos.
- **Does NOT belong here:** deep domain knowledge with discovery triggers → make it a **skill**;
  one-off "what happened" notes → `../sessions/`; future scope → `../roadmap.md`.

Keep each file focused and current. If a note goes stale, fix it — the whole point is that every
session reads the *same* up-to-date reference.
