# sessions — refined session summaries

One file per substantive working session: **`YYYY-MM-DD-<slug>.md`**. Each is a *refined summary*,
not a transcript — just enough that a fresh session resumes with context. Start from
[`_template.md`](_template.md).

## Conventions
- **Name:** `2026-01-15-project-bootstrap.md` (ISO date + kebab-case slug).
- **Length:** aim for ~one screen. Outcomes and current state, not every step taken.
- **Cross-link:** relative-link related sessions; point follow-ups at `../roadmap.md`.
- **Recall newest-first:** dated filenames sort by recency — `ls`/`grep` the dir, then read the
  latest 1–3 or the entry matching the topic. (Pulled on demand, not auto-loaded — see `../README.md`.)
